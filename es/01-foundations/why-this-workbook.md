# Por qué uso GitHub como cuaderno de trabajo

Durante años he usado herramientas para producir:
roadmaps, entregables, soluciones.

Cada vez menos para pensar.

La IA amplifica este problema.
Permite hacer más cosas más rápido,
pero también decidir sin comprender.

Este cuaderno nace como reacción a eso.

## Pensar antes de escalar

En organizaciones grandes, los problemas rara vez vienen
de falta de capacidad técnica.

Vienen de decisiones tomadas demasiado pronto
y escaladas demasiado rápido.

Usar GitHub como cuaderno introduce fricción:
- escribir obliga a ordenar
- publicar obliga a hacerse cargo
- versionar muestra cómo cambian las ideas

No busco respuestas rápidas.
Busco mejores preguntas.

## IA no como herramienta, sino como sistema

La IA no es solo tecnología.
Reorganiza decisiones, responsabilidades y poder.

Cada sistema de IA:
- presupone comportamientos humanos
- define qué es normal y qué es excepción
- desplaza decisiones de personas a sistemas

Esto no se resuelve solo con métricas.

## Cruce con sociología

Empiezo Sociología no para añadir un título,
sino para incomodar mi práctica profesional.

La sociología introduce preguntas que la ingeniería suele evitar:
- quién define el problema
- a quién beneficia la automatización
- qué se pierde cuando optimizamos

Este cuaderno es el lugar donde cruzan
la práctica diaria y esas preguntas.

## Advertencia

Nada de lo que hay aquí es definitivo.
Muchas ideas cambiarán.
Otras se descartarán.

Este no es un repositorio de certezas.
Es un registro de aprendizaje consciente.
